# ?? GUÍA DE INSTALACIÓN Y CONFIGURACIÓN
# Sistema de Gestión Médica - MedManage C#

## ? VERIFICACIÓN PREVIA

Antes de instalar, asegúrese de tener:

### Sistema Operativo
- [ ] Windows 7 o superior
- [ ] Permisos de administrador
- [ ] Mínimo 100 MB de espacio en disco

### Software Requerido
- [ ] .NET Framework 4.7.2 (o superior)
  - Descargar desde: https://www.microsoft.com/es-ES/download/details.aspx?id=56115
- [ ] Microsoft Jet OLEDB 4.0 (incluido en Windows, pero verificar)
- [ ] Visual Studio 2015 o superior (para desarrollo)

### Verificar .NET Framework
```powershell
# Abrir PowerShell y ejecutar:
Get-ChildItem 'HKLM:\SOFTWARE\Microsoft\NET Framework Setup\NDP' -Recurse | 
Get-ItemProperty -Name Version -EA 0 | 
Where {$_.PSChildName -match '^(?!S)\w'} | 
Select PSChildName, Version
```

## ?? INSTALACIÓN

### Opción 1: Visual Studio

1. **Abrir Solución**
   - Visual Studio ? Archivo ? Abrir ? Proyecto/Solución
   - Seleccionar `pryRomoPruebaCopilotAI.sln`

2. **Restaurar Paquetes NuGet**
   - Herramientas ? Administrador de Paquetes NuGet ? Restaurar Paquetes

3. **Compilar**
   - Compilar ? Compilar Solución (Ctrl + Shift + B)
   - Verificar que no hay errores en la ventana de Errores

4. **Ejecutar**
   - Depuración ? Iniciar Depuración (F5)
   - O hacer clic en el botón ? de la barra de herramientas

### Opción 2: Línea de Comandos

```bash
# Navegar a la carpeta del proyecto
cd C:\ruta\pryRomoPruebaCopilotAI

# Compilar con MSBuild
"C:\Program Files (x86)\Microsoft Visual Studio\2019\Community\MSBuild\Current\Bin\MSBuild.exe" pryRomoPruebaCopilotAI.csproj /p:Configuration=Release

# El ejecutable se genera en: bin\Release\pryRomoPruebaCopilotAI.exe
```

### Opción 3: Ejecutable Directo (Después de compilar)

1. Navegar a: `bin\Debug\` o `bin\Release\`
2. Ejecutar: `pryRomoPruebaCopilotAI.exe`
3. Se crea automáticamente la carpeta `Base de Datos\` con `Consultorio.mdb`

## ?? SOLUCIÓN DE PROBLEMAS

### Problema: "No se encontró .NET Framework 4.7.2"
**Solución:**
```
1. Ir a: https://www.microsoft.com/es-ES/download/details.aspx?id=56115
2. Descargar e instalar .NET Framework 4.7.2
3. Reiniciar la computadora
4. Intentar ejecutar nuevamente
```

### Problema: "Error de conexión a la base de datos"
**Soluciones:**
```
1. Verificar permisos: 
   - Clic derecho en la carpeta del ejecutable
   - Propiedades ? Seguridad ? Editar ? Permisos Totales
   
2. Eliminar carpeta "Base de Datos" y ejecutar nuevamente
   
3. Desinstalar y reinstalar el proveedor Jet OLEDB:
   - Descargar desde: 
     https://www.microsoft.com/download/details.aspx?id=23734
```

### Problema: "Error al compilar: No se puede encontrar AccesoDatos"
**Solución:**
```
1. Visual Studio ? Compilar ? Limpiar Solución
2. Visual Studio ? Compilar ? Compilar Solución
3. Si persiste: Eliminar carpeta bin\ y obj\
4. Intentar nuevamente
```

### Problema: "DataGridView vacío"
**Soluciones:**
```
1. Verificar que existen especialidades registradas
2. Crear datos de prueba primero
3. Revisar consultas SQL en la ventana Salida (Debug)
4. Verificar conexión con el botón de estado
```
